Background

This project aims to build the server­side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Get a user GET req=/[users]/[:username] Description : This endpoint retrieves all information about a user as selected by their unique username and returns this in a JSON object format. A similar query can be made using a user's ID /[users]/[:id]. Example return: based on /users/JohnDoe123 { id : '823762', username : 'JohnDoe123', name: 'John Doe', dob : '12/12/12', password :'JohnDoe123', email : 'JohnDoe123@johndoe.com', FavouriteMovies: [...] } Success : 200 / Error : 404/500
Add a user POST req=/[users] Description : This endpoint posts a set of information to create a user. Upon user creation the module UUID assigns each new user a unique identification number which is assigned to a user's "id". A POST request must follow the below JSON format, and MUST include a username, password, email and DOB address for a user. Example post request: based on /users { username : 'JohnDoe123', dob : '12/12/12', password :'JohnDoe123', email : 'JohnDoe123@johndoe.com', FavouriteMovies: [...] } returns : { id : '823762', username : 'JohnDoe123', name: 'John Doe', dob : '12/12/12', password :'JohnDoe123', email : 'JohnDoe123@johndoe.com', FavouriteMovies: [...] } Success : 201 / Error : 422- incorrect format of data entered, 400 - user exists, 500 - error
Validate a user POST req=/[login] Description : This endpoint verifies a users login credentials, and creates a unique token so they can access and interact with the api safely. Example post request: based on /login { username : 'JohnDoe123', password :'JohnDoe123', } returns : { { id : '823762', username : 'JohnDoe123', name: 'John Doe', dob : '12/12/12', password :'JohnDoe123', email : 'JohnDoe123@johndoe.com', FavouriteMovies: [...] }, token : '873406458129474075895' } Success : 201 / Error : 500
Delete a user DELETE req=/[users]/[:username] Description : This endpoint deletes all information about a user as selected by their username and returns a text response confirming that the user has been deleted. Example delete request: based on /users/JohnDoe123 Delete {username : JohnDoe123} return: "User: JohnDoe123 was deleted." Success : 200 / Error : 404/500
Update an existing user PUT req=/[users]/[:username] Description : This endpoint selects an existing user via their username, it then allows users to change fields via a JSON object (see below). Users can update a number of fields at once, and must follow the same keys as provided upon registration (e.g. username). The only field that CANNOT be changed is their "id" number as this is used to prevent multiple user conflicts. User 823762's information for reference { id : '823762', username : 'JohnDoe123', name: 'John Doe', dob : '12/12/12', password :'JohnDoe123', email : 'JohnDoe123@johndoe.com', FavouriteMovies: : [...] } Example put request: based on /users/823762 { 'username' : 'JaneDoe123', 'email' : 'janedoe@janedoe.com' } Expected return: { id : '823762', * username : 'JaneDoe123', name: 'John Doe', dob : '12/12/12', password :'JohnDoe123', * email : 'janedoe@janedoe.com', FavouriteMovies: [...] } Success : 201 / Error : 422 - validation error, 500 - error
Add a movie to favourits POST req =/[users]/[:username]/[Movies]/[:movie_title] Description : This endpoint first selects a user from the database using their unique identification number, this then allows users to add [post] new movies selected by movie_title (also able to select by movie_id) to their favourites list stored under their user id. They can do so by searching movies via their title (in text format). Users are then expected to be returned a text response containing the movie information if successfully updated. Users can then check their list to see the corresponding movie id (and corresponding information can be acquired through this) added to their favourites. For example user/823762/favourite_movies POSTs a request to add "The Beach" post(user/823762/favourite_movies/The%20Beach) This should return a text response stating that "This movie The Beach has been added to your list of favourites!!" Success : 201 / Error : 404/500
Delete a movie from favourits POST req =/[users]/[:username]/[Movies]/[:movie_title] Description : This endpoint first selects a user from the database using their unique identification number, and finds their corresponding "favourites" list. At this point users can send requests to [delete] movies from their list. They can do so by searching movies via their title (in text format) and sending out a delete request. Users are then expected to be returned a text response containing a successful deletion message (see below). Users can then check their favourites list to see the corresponding movie object has been deleted. For example user/823762/favourite_movies/ requests to DELETE "The Beach" delete(user/823762/favourite_movies/The%20Beach) This should return a text response stating that "This movie The Beach has been deleted." Success : 200 / Error : 404/500
