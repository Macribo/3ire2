Background

This project aims to build the server­side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Parameters

Name Description Example
/:movies Description : This endpoint retrieves data on all movies stored (including their: title, description, genres, director, image url). http://localhost:8080/movies
/:movies/:title Description : This endpoint retrieves all data about one specific movie selected by its title name (including: title, description, genres, director, image url) http://localhost:8080/movies/The%20Dark%Knight
/:directors/:name Description : This endpoint retrieves all data about one specific director selected by their name (including: name, bio, dob) http://localhost:8080/directors/Christopher%20Nolan
/:genres/:name Description : This endpoint retrieves all data about one specific genre selected by its name (including: name & description) http://localhost:8080/genres/Comedy
:directors/:name Description : This endpoint retrieves all data about one specific director selected by their name (including: name, bio, dob) http://localhost:8080/:directors/Christopher%20Nolan
/:users Description : This endpoint retrieves all users information (id, username, name, dob, password, email and included an array of their favourite films). It also allows users to be added, new user data is required to be of a JSON object format, { id : '823762', username : 'JohnDoe123', name: 'John Doe', dob : '12/12/12', password :'JohnDoe123', email : 'JohnDoe123@johndoe.com', favourites : [ ] } Each time a new user is POST-ed, a random unique id is assigned to them using the UUID module to prevent multiple username related conflicts. http://localhost:8080/users
/:users/:id Description : This endpoint retrieves all information about a user as selected by their unique identification number (id). The result is in JSON object format and includes: id, username, name, dob, password, email and included an array of their favourite films). As each id is unique to a user, many further functions can be achieved at this endpoint. Users can update their information using PUT requests, their information must follow the JSON object format as follows: { id : '823762', username : 'JohnDoe123', name: 'John Doe', dob : '12/12/12', password :'JohnDoe123', email : 'JohnDoe123@johndoe.com', } This endpoint can also be used to delete user information using (DELETE) requests, and the users unique identification number (id) as a selection parameter to enable accurate de-deregistration. http://localhost:8080/users/823762
/:users/:id/:favourites/:title Description : This endpoint allows users to both add new movies selected by movie_title to their favourites list (POST) or delete movies from their favourites based on the movie_title. Each movie is presented and stored as a JSON object as fokkows: { title : 'The Beach', description: 'Vicenarian Richard travels to Thailand and finds himself in possession of a strange map. Rumours state that it leads to a solitary beach paradise, a tropical bliss. Excited and intrigued, he sets out to find it.', genres: 'Adventure, Drama, Romance, Thriller', director : 'Danny Boyle', image : 'https://m.media-amazon.com/images/M/MV5BNTljZTUzYWUtOTI4YS00NmZlLWE5MmQtZjFlZDZhNjg4MjQxXkEyXkFqcGdeQXVyNTc1NTQxODI@._V1_.jpg', } http://localhost:8080/users/823762/favourites/The%20Dark%20Knight
